Notes de mise à jour actuelles

Moodle 1.4.3 (17 décembre 2004)

(Cette version comporte d'importantes corrections concernant la sécurité. Pour cette raison, nous recommandons vivement que les sites utilisant une version antérieure (quelle qu'elle soit) effectue la mise à jour vers cette version au plus vite.)

    Quelques importantes corrections concernant la sécurité
    • Amélioration de la vérification et du nettoyage des données en cookie
    • Mesures préventives pour la détermination des sessions
    • Amélioration de la vérification et du nettoyage des paramètres des fils RSS, de l'atlas IP, des glossaires, des forums, de la sélection des thèmes, du module SCORM, et du visionneur de documents
    • Ajout d'une protection lors de la visualisation de fichiers déposés
    • Ajout d'une protection pour éviter le passage d'un dossier à un autre
    • Correction pour la variable $FULLME nécessaire dans les crons
    • Prévention contre l'appel direct des librairies internes
    • Interdiction de la visualisation des glossaires cachés
    • Pour les étudiants, les sections cachées sont aussi invisibles dans les activités récentes
    • Correction d'un problème qui permettait aux utilisateurs anonymes de voir les événements du calendrier
    • Correction de l'encodage des expressions TeX
    Quelques nouveautés
    • L'unicité des inscriptions est maintenant exigée. Ceci contribue de plus en une amélioration des performances
    • Ajout du support des transactions pour PostgreSQL
    • Les connexions et déconnexions sont maintenant enregistrées dans les historiques
    • les modules Wiki et SCORM utilisent maintenant un nouveau code pour l'archivage (zip), ce qui corrige certains problèmes rencontrés avec les SCORM
    • Nouveau système de pagination pour les glossaires
    • Les informations des utilisateurs LDAP sont tronquées avant leur insertion dans Moodle afin d'éviter des pertes de données
    • Programme pclzip mis à jour
    • La variable Memory_limit peut maintenant être modifiée dans php.ini, commandline, httpd.conf et .htaccess
    • Nouvelles traductions commencées : Maori, Kannada, Albanais et Vietnamien
    • Nombreux ajouts dans d'autres langues
    • Les enseignants peuvent maintenant essayer les tests même lorsqu'ils ne sont pas encore ouverts
    • La vue des glossaires pour impressions est maintenant différente pour chacun des formats
    • La recherche dans les glossaires trouve maintenant aussi les termes associés
    • Le filtre multilingue a été grandement amélioré, et est plus tolérant avec une syntaxe défaillante
    Améliorations de performance
    • Le traitement des entêtes en cache a été amélioré sur toutes les pages lors de l'utilisation du bouton Retour du navigateur
    • Des index ont été ajoutés en de nombreux endroits de la base de données pour améliorer la performance lors de déploiements à grande échelle
    • Amélioration de performance partout où une liste de cours est affichée
    • Diverses optimisations en retirant des requêtes non nécessaires des bases de données et des tris
    Autres bogues corrigés
    • Correction de la génération d'identifiants de cours doubles lors de la restauration dans un nouveau cours
    • Correction de nombreux petits problèmes avec Postgres
    • Suppression de nombreuses notices non nécessaires en mode deboguage
    • Suppression de notices PHP5 inutiles
    • Correction du support des apostrophes doubles, simples et des barres obliques inverses dans les données des utilisateurs LDAP
    • Correction de la pagination des résultats de recherche dans les cours avec Postgres
    • Correction d'un bogue survenant avec les espaces dans la recherche des utilisateurs
    • Correction d'un bogue dans le chat, qui voyait tous les utilisateurs du chat mentionnés dans plusieurs salons
    • Correction d'un bogue dans le script d'installation
    • Correction du traitement des noms d'utilisateur sensibles à la casse lors de l'importation d'utilisateurs en lots
    • Correction d'un bogue de redirection après évaluation des messages des forums
    • Correction d'un bogue des tests dans les questions d'appariement
    • Correction d'un bogue lors de la prévisualisation des tests
    • Correction d'un bogue lors de l'édition de langues
    • Diverses petites corrections dans le module SCORM
    • Le module SCORM affiche correctement les images des utilisateurs
    • Les activités sont triées par nom dans le filtre d'activités
    • La recherche dans les glossaires trouve maintenant aussi les termes associés
    • Les erreurs RSS génèrent un fichier d'erreur RSS valide plutôt qu'un fil RSS non valide
    • Correction d'un problème de validation de paquets SCORM sous PHP5
    • Correction d'un problème avec les catégories vides lors des sauvegardes et restaurations
    • Meilleur XHTML dans le wiki
    • Correction d'un bogue dans les rapports de chat pour la première session d'un chat
    • Correction d'un problème avec les SCORM lors de la suppression de nombreux ou très gros fichiers
    • Correction de divers problèmes avec le format « Encyclopédie » des glossaires
    • Correction d'un bogue d'intégration de SCORM et du code d'archivage (zip)
    • Correction d'un bogue dans les notifications par courriel format texte des dialogues
    • Ajout des clefs de sessions dans le bloc Activités sociales
    • Les sections en trop d'un cours n'apparaissent plus dans le menu de navigation
    • Corrections pour l'heure d'été dans le calendrier
    • Une catégorie par défaut est assignée de la première édition d'un test dans un nouveau cours
    • Correction d'un bogue dans le module PayPal lors de l'acceptation d'un montant
    • Correction d'un bogue dans les cours : choix impossibles

 

Moodle 1.4.2 (5 novembre 2004)

(Cette version contient d'importants correctifs de sécurité. Pour cette raison, nous recommandons fortement aux administrateurs de sites utilisant une version antérieure de Moodle d'en faire la mise à jour au plus vite !)

    Quelques améliorations importantes de sécurité
    • Améliorations de la vérification et du nettoyage des paramètres des scripts utilisés à de nombreux endroits dans Moodle (grands mercis à Petr Skoda pour son récent audit de sécurité !)
    • Ajout de guillemets à certains paramètres SQL du module Glossaire pour prévenir des injections possibles
    Quelques petites nouvelles fonctionnalités
    • Nouveau module d'authentification PAM permettant l'authentification directe sur Unix/Linux
    • Les responsables de cours peuvent maintenant restaurer les cours dans de nouveaux cours ou dans des cours existants
    • Les messages des forums incluent maintenant un en-tête détaillé pour permettre un suivi correct des sujets dans les clients de courriel
    Et un paquet d'autres corrections de bogues
    • La sauvegarde s'en sort mieux en cas d'erreurs (par exemple si elle est interrompue) et peut nettoyer les fichiers incomplets
    • Correctif pour l'authentification à partir d'une base de données PostgreSQL
    • Correctifs pour certaines parties de l'installation avec une base de données PostgreSQL
    • Les lignes de sujets des forums ne sont maintenant séparées que dans l'affichage des activités récentes
    • Maintenant, l'inscription par l'intermédiaire d'une base de données fonctionne réellement :-)
    • Correctifs SQL pour la création des canaux RSS à partir de PostgreSQL
    • Ajout du protocole rtsp à la liste des protocoles autorisés dans le texte HTML
    • Les profils utilisateurs et la page de modification du profil peuvent être appelés sans paramètres
    • Mise à jour de la librairie xmlize à sa dernière version
    • Mise à jour de la librairie phpmailer à sa dernière version
    • Divers changements cosmétiques dans le code
    • Le menu Administration permet maintenant la sauvegarde et la restauration du « cours de site » (page d'accueil)
    • Correction d'un bogue d'affichage dans l'index des forums (n'apparaissant que sous PHP5)
    • La fonction de mise en surbrillance (par exemple dans les résultats de recherche) supprimait les barres obliques inverses
    • L'information en provenance de LDAP est maintenant décodée en utf8
    • Les listes de cours fonctionnent mieux, lorsqu'il y a un très grand nombre de cours
    • Deux bogues corrigés dans le module SCORM (une faute de frappe et un problème avec onbeforeunload)
    • Quelques améliorations cosmétiques lors de l'impression des textes, des en-têtes et des pieds de page dans le module Leçon
    • Le calendrier fonctionne maintenant correctement, même si l'on se connecte sous un autre nom
    • Correction de quelques balises PHP abrégées restantes
    • Mélange plus aléatoire des questions aléatoires (correction d'un bogue introduit dans la version 1.4)
    • Quand des enseignants sont retirés d'un cours, ils sont également retirés de tous les groupes de ce cours
    • Correction de strip_pages dans le module Wiki
    • Corrections dans la détection du jeu de caractères dans les sauvegardes et le filtre multilingue, sous PHP5

 

Moodle 1.4.1 (12 septembre 2004)

    Quelques petites nouvelles fonctionnalités
    • Nouveau filtre « Protection adresses de courriel » qui cache toutes les adresses de courriel dans tous les textes. Cela évite la collection des adresses de courriel par des robots
    • Possibité de restreindre les adresses de courriel permises (voir le fichier « config-dist.php » pour des détails)
    • Instructions supplémentaires pour les traducteurs, intégrées dans l'interface de traduction
    • Nouvelle page d'aide pour les chemins d'accès
    Et quelques bogues corrigés
    • Amélioration de la performance lors de l'affichage des historiques
    • Amélioration de la pagination et de la performance dans les glossaires
    • Les articles des glossaires sont affichés correctement même s'ils appartiennent à plusieurs catégories
    • Pas de redéclaration de la variable auth_user_login lors des échecs de connexion
    • Correction de deux icônes manquantes dans le thème « cordoroyblue »
    • Le bouton manquant « Nouvel événement » a été ajouté sur la page des prochains événements
    • Ajout systématique de addslashes dans les messages du chat (avec le démon) avant leur incorporation dans la base de données
    • Corrections pour le traitement des périodes d'inscriptions fixes, lorsque cette option est activée dans un cours
    • On ne suppose plus que l'identifiant du cours de site (défini comme SITEID) est toujours 1
    • La limite de mémoire par défaut a été augmentée au delà de 16Mo, pour permettre à Moodle de continuer de fonctionner même lorsque des problèmes de mémoire surviennent inopinément
    • Les binettes sont maintenant transformées en images dans les textes en format Markdown
    • Les ateliers apparaissent de nouveau dans les activités récentes
    • Le fichier file.php ne peut maintenant être utilisé que pour accéder aux données à l'intérieur des cours
    • Meilleur comportement du bloc de calendrier sur la page d'accueil
    • Correction du bogue des guillemets avec la spécification des bases de données PostgreSQL produite par l'installeur
    • Correction de l'affichage parfois déficient des réponses correctes durant la relecture des tests
    • Les balises <tt> et <code> sont maintenant autorisées
    • Le code JavaScript dans les résumés des sections, des cours et les étiquettes n'est plus filtré
    • Correction d'un rare bogue LDAP pouvant causer la non-création de nouveaux comptes

 

Moodle 1.4 (31 août 2004)

    Manchette
    • Compatibilité avec PHP 5
    • Nouvel installeur facile à utiliser, qui permet la création du fichier config.php initial
    • L'installation est maintenant entièrement traduite dans votre langue
    • Module « Ressources » entièrement réorganisé pour rendre plus facile l'ajout de nouvelles ressources et l'envoi de données aux ressources
    • Nouveau système d'inscription aux cours, utilisant des modules, par exemple un système de paiement Paypal automatisé, le traitement d'un fichier texte ou le contrôle d'accès à partir d'une base de données externe
    • Les blocs fonctionnent maintenant sur la page d'accueil, et il est possible d'ajouter une section thématique au-dessus de la partie centrale de cette page
    • Le module « Chat » inclut maintenant un démon (en PHP, côté serveur) permettant des discussions rapides et extensibles
    • Nouveau module « Wiki » !
    • Support du format Markdown pour l'édition des textes
    • De nombreuses corrections de bogues dans tous les domaines
    Sous le capot
    • Le module « Ressources » possède son propre menu Ajouter. Il utilise des sous-modules basés sur des classes, rendant plus facile l'ajout de nouveaux types de ressources à ce menu
    • Le module « Test » a été reorganisé en modules de types de question, basés sur des classes
    • Le processus d'inscription aux cours a été réorganisé en modules d'inscription, basés sur des classes
    • Les formats d'affichage des « Glossaires » sont maintenant basés sur des noms, et il est ainsi plus facile d'en ajouter de nouveaux
    • Fonctionnement amélioré de nombreux modules lors de leur utilisation sur la page d'accueil du site
    • Structure des filtres révisée, plus rapide et maintenant compatible avec Turck MMcache
    • L'ajout d'activités est moins strict lorsque certains champs sont laissés vides
    • On peut maintenant modifier les colonnes des tables PostgreSQL à l'aide de table_column()
    • La génération RSS a été accélérée de manière substantielle, et surcharge beaucoup moins le cron
    • Certains gros modules ont leur fichier lib.php séparé en locallib.php pour diminuer l'utilisation de la mémoire
    Langues et localisation
    • Presque toutes les langues ont été mises à jour
    • Moodle peut remplacer le jeu de caractères par défaut fixé par Apache, ce qui rend les langues plus fiables
    • Les textes des langues codées sur plusieurs octets reviennent à la ligne correctement
    • Meilleur traitement des réglages locaux (locale) et des encodages sur les sites très multilingues (comme moodle.org)
    • Nouvelles langues : slovène et hébreux !
    Sécurité
    • Quelques entrées XSS possibles ont été fermées et quelques divulgations de chemins supprimées
    • Les attributs des balises HTML sont maintenant traités beaucoup plus exhaustivement, pour se défendre des scripts délétères
    • Les pages des profils utilisateur ne sont par défaut plus accessibles aux étrangers
    • Tout le texte est maintenant nettoyé lors de la sortie (output), et plus seulement à l'entrée (input)
    • Les adresses de courriel ne sont plus utilisées sur les messages sortant, si l'utilisateur le demande
    • Un nouveau réglage loginhttps permet d'utiliser le protocole sécurisé https pour la connexion, et de repasser ensuite en protocole http
    Administration
    • Nouvelles interfaces pour l'ajout et le retrait d'étudiants, d'enseignants, de créateurs de cours et d'administrateurs
    • Les échecs de connexion sont enregistrés et peuvent être affichés aux utilisateurs lors d'une prochaine connexion valable
    • Les historiques permettent l'affichage par groupe et par type d'erreur
    • Les modules fonctionnent maintenant de façon plus cohérente sur la page d'accueil du site
    • Les « étudiants » des activités de la page d'accueil peuvent être soit tous les utilisateurs du site, soit les utilisateurs inscrits à au moins un cours
    • L'envoi de courriels peut être activé et désactivé directement dans le profil des utilisateurs
    • Les visiteurs peuvent être automatiquement connectés en tant qu'utilisateurs anonymes
    • Le menu « Aller à... » affiche toujours le texte « Aller à », pour indiquer plus clairement à quoi il sert
    • Le menu « Aller à... » regroupe de façon bien plus claire les activités en sections
    Authentification
    • Toutes les méthodes d'authentification coexistent avec les comptes créés en interne
    • Nouveau support de l'authentification par les serveurs First Class
    • Correctifs pour l'authentification par POP3
    Blocs
    • Le système de blocs s'accommode des formats de cours personnalisés et des changements de format de cours
    • Un nouveau bloc de connexion permet la connexion depuis la page d'accueil du site
    • Le bloc « Utilisateurs en ligne » affiche sur la page d'accueil les utilisateurs connectés du site
    Éditeur WYSIWYG
    • Nouvelle page de configuration de l'éditeur pour l'administrateur
    • Support des ancrages à l'intérieur des pages
    • La correction orthographique est supportée dans l'éditeur. Elle requiert le fonctionnement du logiciel « aspell » sur votre serveur
    Chat
    • Le module « Chat » inclut maintenant un démon (en PHP, côté serveur) permettant des discussions rapides et extensibles
    Sondage
    • Remanié de façon significative pour améliorer le fonctionnement, avec plus d'options disponibles
    Forum
    • Divers correctifs lors de l'utilisation des groupes
    • Les utilisateurs peuvent choisir de recevoir les messages en résumés journaliers (sujets des messages ou messages complets)
    • Les listes des discussions sont maintenant paginées
    • Les enseignants ont maintenant un contrôle total sur les abonnements aux forums, et peuvent abonner n'importe qui à volonté
    • La date du dernier message est maintenant abrégée et liée au dernier message
    • La page d'index des forums indique les canaux RSS disponibles
    • Les courriels en format HTML sont plus jolis, grâce à l'utilisation d'une feuille de style
    • Les messages des forums vieux de plus de deux jours ne sont plus envoyés par courriel, pour éviter des engorgements de courriels non désirés
    • Les courriels bloqués sont enregistrés, afin que les enseignants puissent les voir
    • Les fils de discussions peuvent être coupés de la discussion principale, pour créer une nouvelle discussion
    Glossaire
    • Les articles de glossaire peuvent être définis comme toujours modifiables
    • La page d'index des glossaires indique les canaux RSS disponibles
    • De nombreux correctifs
    Test
    • Nouveau chronomètre (flottant) pour les tests avec durée limitée
    • Les tentatives peuvent être restreintes à certains ordinateurs spécifiques
    • Les tests peuvent maintenant demander un mot de passe
    • Nouveau type de question : Question calculée
    • Les questions numériques supportent maintenant les unités
    • Les questions peuvent être prévisualisées dans les pages d'édition
    • Les questions peuvent être exportées en divers formats
    • Traitement plus intelligent de la liste des questions par catégorie lors de leur édition
    • Nouveau format d'exportation XML (encore incomplet), englobera ultérieurement toutes les fonctionnalités des tests
    • Les réponses courtes peuvent maintenant contenir toutes sortes d'étranges caractères
    Ressource
    • L'ajout de ressources se passe maintenant en une étape, avec plus de contrôle sur les options
    • Les types de ressource Page web, Lien web, Fichier déposé et Programme externe ont été combinés en un seul type !
    • Les ressources de type Programme sont remplacées par des paramètres permettant une grande interaction avec des ressources externes, qu'elle soient hébergées sur d'autres serveurs ou intégrées à Moodle.
    • Des formulaires escamotables permettent de rendre l'interface moins complexe pour les débutants, sans handicaper les utilisateurs chevronnés
    SCORM
    • De nombreux correctifs et améliorations. Le module fonctionne avec une éventail encore plus large de paquets SCORM
    • Vérification supplémentaire optionnelle du fichier « manifest »

 

N'oubliez pas de jeter un coup d'oeil aux nombreux modules optionnels innovateurs en développement.

Si vous découvrez des bogues dans cette version, merci de nous les annoncer sur moodle.org/bugs, afin que nous puissions les corriger dans la prochaine version.
