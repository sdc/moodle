Anciennes notes de mise à jour

Moodle 1.4.5 (7 mai 2005)

(Cette version comporte d'importantes corrections concernant la sécurité. Pour cette raison, nous recommandons vivement aux responsables de sites utilisant une version antérieure, quelle qu'elle soit, d'effectuer au plus vite la mise à jour vers cette version.)

Quelques importantes corrections concernant la sécurité
  • Mesures préventives contre les attaques contre les fonctions d'administration.
  • Leçons : masquage plus sûr des leçons (bogue 2733).
Quelques nouveautés
  • Lecteur multémédia Flash : la lecture peut commencer avant la fin du téléchargement de la séquence.
  • Ajout/mise à jour des langues allemande (DE) et turque (TR).
  • Les notes de mise à jour indiquent maintenant les liens vers les bogues concernés.
  • Les requêtes anticipées (prefetching requests) de Google Web Accelerator, Mozilla, etc. sont repoussées.
Améliorations de performance
  • Ajout d'une option permettant la mise en cache de la liste des langues. Cette nouveauté permet d'économiser environ 10 Mo de mémoire et 40 inclusions de fichier. Voir le fichier config-dist.php pour les details de l'activation.
  • Améliorations majeures de la performance des insertions avec les bases de données Postgres. Correction de nombreux appels de la fonction insert_record() n'utilisant pas l'identifiant de l'enregistrement (id) pour ne pas en faire la demande.
Corrections d'autres bogues
  • Éditeur WYSIWYG : correction de quelques problèmes d'encodage.
  • Test : correction au menu déroulant d'exportation/importation (bogue 2963).
  • Test : n'abandonne pas lors d'une erreur sur un question inconnue, essaie en revanche de continuer.
  • Forum d'un cours informel : suppression d'un lien trompeur pour se désabonner (bogue 2945).
  • Traductions supplémentaires/augmentées.
  • Forums et groupes : les étudiants ne faisant partie d'aucun groupe ont l'autorisation d'écrire des messages dans des discussions ne faisant pas non plus partie de groupes, même si le cours utilise des groupes.
  • Réglages globaux de l'éditeur WYSIWYG : correction du bogue Impossible de modifier la configuration de l'éditeur.
  • Cron : suppression de quelques avertissements.
  • Sondage : sondage donnant un faux nombre de réponses (bogue 2452).
  • Meilleur support de PHP5 : le dépôt de fichiers utilise maintenant la variable $_FILES au lieu de $HTTP_POST_FILES (bogue 2828).
  • Sauvegarde/restauration : les liens manuels des pages des leçons vers des ressources sont maintenant partout préservés entre les sauvegardes/restaurations. Voir cette discussion.
  • Quelques corrections concernant le nommage en Postgres.
  • Support internationalisation : l'encodage adéquat est correctement fixé lors de la connexion (bogue 2770).
  • Support internationalisation : plusieurs correctifs pour des caractères utf-8 brouillés par l'éditeur WYSIWYG (bogue 2763)
  • Lecteur multémédia Flash : le chargement automatique des fichiers audio est désactivé.
  • Lecteur multémédia Flash : meilleur traitement des noms de fichier.
  • Postgres : correction de séquences dans un ordre erroné (bogue 2742).
  • Sauvegarde : correction du dépassement de la limite de mémoire dans la page de configuration.
  • Forums : correction d'un message d'avertissement lorsqu'un enseignant poste un message dans un forum (bogue 2723).

 

Moodle 1.4.4 (7 mars 2005)

Quelques importantes corrections concernant la sécurité
  • Ajout de robustes fonctions de validation d'URL
  • Mesures préventives contre la modification malicieuse de la configuration
  • Validation plus stricte des données saisies dans les tests, l'éditeur WYSIWYG et l'édition des cours
  • Meilleur contrôle des réglages de langue des utilisateurs
  • Nettoyage plus sûr des textes saisis en HTML
Quelques nouveautés
  • Nouveau filtre MP3 avec un lecteur mp3 en Flash
  • Meilleur affichage des options dans le module Sondage
  • Améliorations de l'interface de l'éditeur de tables dans l'éditeur WYSIWYG
  • Les réglages $CFG du fichier config.php ont maintenant priorité sur ceux de la base de données
  • Activation de la récupération de place (garbage collection) pour les systèmes Debian, avec la possibilité de la désactiver. Ceci permet de corriger des problèmes de session figée
Améliorations de performance
  • Sérieuses optimisations de performance dans le code des forums
  • Diverses optimisations, grâce à la suppression de requêtes et de tris de base de données
  • Améliorations majeures du traitement des verrous de session. Le téléchargement de gros fichiers ou la consultation en direct des historiques ne ralentit plus la navigation dans d'autres fenêtres
Corrections d'autres bogues
  • Installeur : nombreuses corrections et mises à jour tirées du tronc (y compris le bogue 2625)
  • Installeur : une vérification est effectuée sur allow_url_fopen() avant d'essayer de valider l'URL de l'installation
  • Correction de certains problèmes dans la recherche des forums
  • Correction d'un bogue de tri sur les forums (bogue 2613)
  • Correction de problèmes dans la recherche des utilisateurs avec des espaces finaux
  • Correction de quelques chemins vers /admin codés en dur
  • Correction de quelques bogues en relation avec les groupes (bogues 2133, 2550, 2363)
  • Moodle n'envoie plus des résumés en HTML aux personnes désirant des courriels en texte pur
  • Correction d'une erreur de la barre de navigation dans le module SCORM
  • Chat : la taille maximale de lecture de l'interface de connexion (socket) a été augmentée, afin d'éviter les pages vides et les répétitions de messages
  • Fichiers : ajout d'une icône pour le dossier parent (bogue 2491)
  • Support de Postgres : correction de la cohérence dans la table mdl_log
  • Plugins d'authentification : meilleur nettoyage et troncage des données
  • Plugins d'authentification : correction d'un problème d'authentification externe — un utilisateur connecté comme anonyme ne pouvait se connecter par une authenfication externe lors de la toute première connexion avec un compte
  • Authentification LDAP : correction d'une régression depuis 1.4.2, où un point-virgule final dans certains champs empêchait l'authentification
  • Authentification par base de données : cette authentification n'allait pas chercher correctement les champs de données de l'utilisateur (bogue 2319)
  • Plugin d'inscription PayPal : le texte envoyé à PayPal peut maintenant être nettoyé pour éviter certaines marques diacritiques (espagnoles pour l'instant) qui empêchent le déroulement des inscriptions. Actuellement, seules les marques diacritiques françaises et allemandes sont supportées par PayPal. Cette fonctionnalité peut être facilement étendue à d'autres caractères
  • Sauvegarde/restauration : si l'utilisateur n'est pas un administrateur et si le cours restauré ne comporte pas d'enseignant, l'utilisateur effectuant la restauration y est automatiquement ajouté comme enseignant (bogue 2381)
  • Sauvegarde/restauration : erreurs corrigées avec les cours dont le nom comporte des apostrophes
  • Sauvegarde/restauration : les membres des groupes ne sont maintenant inclus (lors de la sauvegarde et de la restauration) que si l'incorporation des utilisateurs a été choisie (bogue 2486)
  • Sauvegarde/restauration : nettoyage général du code
  • Glossaire : la note 0 est maintenant supportée (bogue 2582), et PHP5 est mieux supporté (bogue 2581)
  • Glossaire & Blackboard : corrections dans l'importation (bogue 2310)
  • Calendrier : traitement plus robuste de données erronées
  • Calendrier : corrections dans le code HTML (bogue 2350)
  • L'ordre de tri des cours était défectueux dans la version 1.4.3. Cela perturbait parfois la création des cours. Tous les problèmes connus avec l'ordre de tri sont maintenant corrigés. Quelques corrections ont été également effectuées dans les appels de la fonction get_categories(), qui apparaissaient parfois dans la liste « Mes cours »
  • Wiki : plusieurs corrections
  • La lecture des historiques en direct possède maintenant un pied de page adéquat
  • Divers nettoyages et rangements du code en général

Moodle 1.4.1 (12 septembre 2004)

  • Nouveau filtre « Protection adresses de courriel » qui cache toutes les adresses de courriel dans tous les textes. Cela évite la collection des adresses de courriel par des robots
  • Possibité de restreindre les adresses de courriel permises (voir le fichier « config-dist.php » pour des détails)
  • Instructions supplémentaires pour les traducteurs, intégrées dans l'interface de traduction
  • Nouvelle page d'aide pour les chemins d'accès
Et quelques bogues corrigés
  • Amélioration de la performance lors de l'affichage des historiques
  • Amélioration de la pagination et de la performance dans les glossaires
  • Les articles des glossaires sont affichés correctement même s'ils appartiennent à plusieurs catégories
  • Pas de redéclaration de la variable auth_user_login lors des échecs de connexion
  • Correction de deux icônes manquantes dans le thème « cordoroyblue »
  • Le bouton manquant « Nouvel événement » a été ajouté sur la page des prochains événements
  • Ajout systématique de addslashes dans les messages du chat (avec le démon) avant leur incorporation dans la base de données
  • Corrections pour le traitement des périodes d'inscriptions fixes, lorsque cette option est activée dans un cours
  • On ne suppose plus que l'identifiant du cours de site (défini comme SITEID) est toujours 1
  • La limite de mémoire par défaut a été augmentée au delà de 16Mo, pour permettre à Moodle de continuer de fonctionner même lorsque des problèmes de mémoire surviennent inopinément
  • Les binettes sont maintenant transformées en images dans les textes en format Markdown
  • Les ateliers apparaissent de nouveau dans les activités récentes
  • Le fichier file.php ne peut maintenant être utilisé que pour accéder aux données à l'intérieur des cours
  • Meilleur comportement du bloc de calendrier sur la page d'accueil
  • Correction du bogue des guillemets avec la spécification des bases de données PostgreSQL produite par l'installeur
  • Correction de l'affichage parfois déficient des réponses correctes durant la relecture des tests
  • Les balises <tt> et <code> sont maintenant autorisées
  • Le code JavaScript dans les résumés des sections, des cours et les étiquettes n'est plus filtré
  • Correction d'un rare bogue LDAP pouvant causer la non-création de nouveaux comptes

 

Moodle 1.4 (31 août 2004)

Manchette
  • Compatibilité avec PHP 5
  • Nouvel installeur facile à utiliser, qui permet la création du fichier config.php initial
  • L'installation est maintenant entièrement traduite dans votre langue
  • Module « Ressources » entièrement réorganisé pour rendre plus facile l'ajout de nouvelles ressources et l'envoi de données aux ressources
  • Nouveau système d'inscription aux cours, utilisant des modules, par exemple un système de paiement Paypal automatisé, le traitement d'un fichier texte ou le contrôle d'accès à partir d'une base de données externe
  • Les blocs fonctionnent maintenant sur la page d'accueil, et il est possible d'ajouter une section thématique au-dessus de la partie centrale de cette page
  • Le module « Chat » inclut maintenant un démon (en PHP, côté serveur) permettant des discussions rapides et extensibles
  • Nouveau module « Wiki » !
  • Support du format Markdown pour l'édition des textes
  • De nombreuses corrections de bogues dans tous les domaines
Sous le capot
  • Le module « Ressources » possède son propre menu Ajouter. Il utilise des sous-modules basés sur des classes, rendant plus facile l'ajout de nouveaux types de ressources à ce menu
  • Le module « Test » a été reorganisé en modules de types de question, basés sur des classes
  • Le processus d'inscription aux cours a été réorganisé en modules d'inscription, basés sur des classes
  • Les formats d'affichage des « Glossaires » sont maintenant basés sur des noms, et il est ainsi plus facile d'en ajouter de nouveaux
  • Fonctionnement amélioré de nombreux modules lors de leur utilisation sur la page d'accueil du site
  • Structure des filtres révisée, plus rapide et maintenant compatible avec Turck MMcache
  • L'ajout d'activités est moins strict lorsque certains champs sont laissés vides
  • On peut maintenant modifier les colonnes des tables PostgreSQL à l'aide de table_column()
  • La génération RSS a été accélérée de manière substantielle, et surcharge beaucoup moins le cron
  • Certains gros modules ont leur fichier lib.php séparé en locallib.php pour diminuer l'utilisation de la mémoire
Langues et localisation
  • Presque toutes les langues ont été mises à jour
  • Moodle peut remplacer le jeu de caractères par défaut fixé par Apache, ce qui rend les langues plus fiables
  • Les textes des langues codées sur plusieurs octets reviennent à la ligne correctement
  • Meilleur traitement des réglages locaux (locale) et des encodages sur les sites très multilingues (comme moodle.org)
  • Nouvelles langues : slovène et hébreux !
Sécurité
  • Quelques entrées XSS possibles ont été fermées et quelques divulgations de chemins supprimées
  • Les attributs des balises HTML sont maintenant traités beaucoup plus exhaustivement, pour se défendre des scripts délétères
  • Les pages des profils utilisateur ne sont par défaut plus accessibles aux étrangers
  • Tout le texte est maintenant nettoyé lors de la sortie (output), et plus seulement à l'entrée (input)
  • Les adresses de courriel ne sont plus utilisées sur les messages sortant, si l'utilisateur le demande
  • Un nouveau réglage loginhttps permet d'utiliser le protocole sécurisé https pour la connexion, et de repasser ensuite en protocole http
Administration
  • Nouvelles interfaces pour l'ajout et le retrait d'étudiants, d'enseignants, de créateurs de cours et d'administrateurs
  • Les échecs de connexion sont enregistrés et peuvent être affichés aux utilisateurs lors d'une prochaine connexion valable
  • Les historiques permettent l'affichage par groupe et par type d'erreur
  • Les modules fonctionnent maintenant de façon plus cohérente sur la page d'accueil du site
  • Les « étudiants » des activités de la page d'accueil peuvent être soit tous les utilisateurs du site, soit les utilisateurs inscrits à au moins un cours
  • L'envoi de courriels peut être activé et désactivé directement dans le profil des utilisateurs
  • Les visiteurs peuvent être automatiquement connectés en tant qu'utilisateurs anonymes
  • Le menu « Aller à... » affiche toujours le texte « Aller à », pour indiquer plus clairement à quoi il sert
  • Le menu « Aller à... » regroupe de façon bien plus claire les activités en sections
Authentification
  • Toutes les méthodes d'authentification coexistent avec les comptes créés en interne
  • Nouveau support de l'authentification par les serveurs First Class
  • Correctifs pour l'authentification par POP3
Blocs
  • Le système de blocs s'accommode des formats de cours personnalisés et des changements de format de cours
  • Un nouveau bloc de connexion permet la connexion depuis la page d'accueil du site
  • Le bloc « Utilisateurs en ligne » affiche sur la page d'accueil les utilisateurs connectés du site
Éditeur WYSIWYG
  • Nouvelle page de configuration de l'éditeur pour l'administrateur
  • Support des ancrages à l'intérieur des pages
  • La correction orthographique est supportée dans l'éditeur. Elle requiert le fonctionnement du logiciel « aspell » sur votre serveur
Chat
  • Le module « Chat » inclut maintenant un démon (en PHP, côté serveur) permettant des discussions rapides et extensibles
Sondage
  • Remanié de façon significative pour améliorer le fonctionnement, avec plus d'options disponibles
Forum
  • Divers correctifs lors de l'utilisation des groupes
  • Les utilisateurs peuvent choisir de recevoir les messages en résumés journaliers (sujets des messages ou messages complets)
  • Les listes des discussions sont maintenant paginées
  • Les enseignants ont maintenant un contrôle total sur les abonnements aux forums, et peuvent abonner n'importe qui à volonté
  • La date du dernier message est maintenant abrégée et liée au dernier message
  • La page d'index des forums indique les canaux RSS disponibles
  • Les courriels en format HTML sont plus jolis, grâce à l'utilisation d'une feuille de style
  • Les messages des forums vieux de plus de deux jours ne sont plus envoyés par courriel, pour éviter des engorgements de courriels non désirés
  • Les courriels bloqués sont enregistrés, afin que les enseignants puissent les voir
  • Les fils de discussions peuvent être coupés de la discussion principale, pour créer une nouvelle discussion
Glossaire
  • Les articles de glossaire peuvent être définis comme toujours modifiables
  • La page d'index des glossaires indique les canaux RSS disponibles
  • De nombreux correctifs
Test
  • Nouveau chronomètre (flottant) pour les tests avec durée limitée
  • Les tentatives peuvent être restreintes à certains ordinateurs spécifiques
  • Les tests peuvent maintenant demander un mot de passe
  • Nouveau type de question : Question calculée
  • Les questions numériques supportent maintenant les unités
  • Les questions peuvent être prévisualisées dans les pages d'édition
  • Les questions peuvent être exportées en divers formats
  • Traitement plus intelligent de la liste des questions par catégorie lors de leur édition
  • Nouveau format d'exportation XML (encore incomplet), englobera ultérieurement toutes les fonctionnalités des tests
  • Les réponses courtes peuvent maintenant contenir toutes sortes d'étranges caractères

  • Les images dans /pix ont été réorganisées
  • Les courriels en format HTML sont mieux formatés pour une meilleure compatibilité
  • Les courriels non remis et retournés sont maintenant envoyés à l'administrateur
  • Les tâches cron demandent moins de temps CPU et les inscriptions à doubles sont aussi nettoyées
  • Amélioration de l'affichage des historiques avec un très grand nombre de pages et une souplesse accrue
  • Les historiques des utilisateurs fonctionnent au niveau du site (affichant TOUS les historiques du site)
  • Les utilisateurs ayant un compte manuel peuvent récupérer leur mot de passe
  • Meilleur système pour définir et imprimer les fichiers d'aides des modules d'activité principaux
  • Les modules d'activité ont maintenant des réglages
  • Les adresses de courriel sont maintenant complètement masquées (pour tromper les robots aspirateurs)
  • Il y a maintenant un index des fichiers d'aide
  • Le module supplémentaire Library (de Mark Kimes') est maintenant supporté s'il est installé (voir les contributions à Moodle pour le code source de Library)
Cours
  • On peut maintenant supprimer aux enseignants les privilèges de modification de leurs cours
  • Les enseignants peuvent maintenant inscrire manuellement leurs étudiants au besoin
  • La gestion des enseignants est maintenant regroupée en une interface
  • Les activités peuvent être déplacées n'importe où en deux clics
  • Les thèmes et semaines peuvent être déplacées vers le haut ou le bas
  • Un menu déroulant permet de changer rapidement de thème et de semaine
  • Lorsque les étudiants s'inscrivent, ils reçoivent une notification et peuvent annuler leur inscription
  • Les enseignants et étudiants voient « Mes cours » sur les pages d'accueil des cours
  • Si elles ne sont pas utilisées, les notes peuvent être complètement cachées pour les élèves
Langues
  • Le hongrois et le grec ont été ajoutés, et il y a un grand nombre d'autres mises à jour. Comme d'habitude, les fichiers de langue de cette version ne sont pas tous terminés. Vous pouvez toujours télécharger les derniers fichiers de langue sur http://moodle.org/download/lang/.
  • Mises à jour et corrections de nombreuses langues
  • Les langues ont maintenant une langue parent pour l'affichage des chaînes manquantes
  • Corrections de l'affichage des dates et des temps dans plusieurs langues
  • es_mx a été remplacé par es (espagnol international)
Forums
  • Beaucoup d'améliorations dans la recherche, plusieurs mots-clef, pages, etc.
  • L'évaluation d'un forum peut être active ou non, utilisable par tous ou seulement par les enseignants
  • L'évaluation des forums peut utiliser des barèmes personnalisés ou des nombres
  • Un intervalle de dates peut être fixé pour restreindre les messages pouvant être évalués
  • L'administrateur peut accéder à certains réglages via Admin -> Modules -> Forums
  • Après modification d'un message, l'utilisateur est renvoyé au message dans son contexte
  • Les images en annexe sont affichées en ligne
  • Les copies courriel des messages en format texte affichent maintenant correctement les URLs
Tests
  • Nouveau type de question : format Cloze - permet des textes contenant en ligne les réponses
  • Nouveau type de question : numérique - permet des réponses dans un intervalle de nombres
  • Nouveau type de question : description - n'est en fait pas une question, mais permet d'introduire n'importe où dans un test du texte ou des images
  • Rapport étendu pour les tests, avec des modules de rapport flexibles
  • Nouvelle fonctionnalité pour les tentatives répétées, où chaque tentative complète la précédente.
  • Si le test se termine dans les prochaines 24 heures, un petit compte à rebours est affiché dans la barre de titre et avertit l'utilisateur à 10 minutes et à 0 minutes
  • Modifications du code du module test, afin de simplifier son extension par de nouveaux types de questions
Ressources
  • Les liens web peuvent maintenant être ouvert dans d'autres fenêtres entièrement personnalisables
  • L'administrateur peut accéder à certains réglages via Admin -> Modules -> Ressources
Quelques bogues parmi les plus ennuyeux corrigés
  • Lors du déplacement des messages, les annexes sont aussi déplacées
  • Plus d'avertissements lors de la recherche de fichiers déposés dans des cours vides
  • Les étudiants ne reçoivent plus systématiquement des courriels en format texte des forums dont ils reçoivent obligatoirement une copie des messages.
  • Les cookies ne peuvent plus interférer avec la mise en place du compte de l'administrateur lors de l'installation
  • Plus de limite dans le nombre d'activités par section
  • Plusieurs corrections lors de l'utilisation des fichiers zip, en particulier sous Windows
  • Les administrateurs peuvent maintenant modifier les comptes créés à partir de bases de données externes.
  • Les sessions n'entrent plus en conflit avec d'autres logiciels
  • Les fichiers Excel fonctionnent maintenant aussi sur autre chose que les machines Wintel
 
 

Nouveautés de Moodle 1.0.9 (29 mai 2003) :

Généralités
  • Diverses optimisations et améliorations significatives des performaces
  • Plusieurs obscures corrections pour la sécurité
  • Moodle se comporte correctement quand il est incorporé dans un cadre
  • L'éditeur Richtext peut être maintenant utilisé dans plus de champs texte
  • Nouveau format « texte », utile pour taper du code HTML
  • Nouveau format « genre Wiki », permet l'ajout de balises sur le modèle Wiki à de nombreux endroits
  • Nouvelle fenêtre permettant d'insérer des binettes en cliquant sur des images
  • Possibilité d'indiquer les permissions des nouveaux dossiers et fichiers créés sur le serveur
  • Quelques corrections pour les serveurs tournant en Safe Mode (quelques problèmes de fichiers subsistent)
  • Diverses corrections pour le nettoyage des textes externes de caractères comme l'apostrophe, etc.
  • Diverses corrections pour compatibilité avec PostgreSQL 7.3
  • Début de transition du code HTML vers XHTML Transitional
  • D'innombrables autres petites corrections un peu partout
Langues
  • 10 nouvelles langues ! Espagnola argentin, Tchèque, Portugais, Slovaque, Roumain, Danois, Russe, Polonais, Chinois traditionnel et Canadien français.
  • Corrections et mises à jour pour presque toutes les langues
  • La langue peut être choisie pour la session en cours (voir le menu sur la page d'accueil et sur la page de connexion) et peut être changée pour chaque page à l'aide de l'URL, p. ex. http://moodle.org/?lang=ru
  • Le menu des langues peut maintenant être retreint à certaines langues, et même retiré
  • Des polices TrueType sont maintenant intégrées aux langues
  • TOUTES les heures et dates peuvent maintenant être localisées dans les fichiers de langue
  • Un bogue petit, mais ennuyeux a été corrigé dans l'éditeur des langues, qui causait sur les serveurs Windows un grossissement des fichiers de langue.
Développeurs
  • Une variable de débogage globale donne plus d'informations sur les variables non initialisée et d'autre paramètres analogues
  • Diverses corrections pour nettoyer et standardiser le code PHP
  • Diverses corrections pour plus de robustesse (assurance que les variables sont toujours initialisées)
Administration
  • Plusieurs options et quelques morceaux de code de config-dist.php ont été modifiés (en cas de mise à jour, comparez-le avec votre config.php ou reconstruisez complètement votre config.php)
  • Nombreuses modifications dans les thèmes - les auteurs de thèmes doivent lire theme/UPGRADE.txt
  • Nouvelle fonction d'enregistrement sur moodle.org permettant l'envoi de bulletin de sécurité, etc.
  • Le dossier admin peut être déplacé ailleurs que /admin (voir config-dist.php)
  • Les listes d'utilisateurs sont maintenant paginées et l'on peut rechercher des chaînes de caractères dans la liste des utilisateurs
  • Gestion des modules : les modules d'activité peuvent maintenant être désactivés individuellement ou effacés complètement
  • Amélioration dans la page d'édition des langues
  • Meilleure vérification de l'environnement PHP durant l'installation, avec notification d'avertissements
  • Procédure d'installation plus propre
Gestion des utilisateurs
  • Nouveau rôle : Responsable de cours - un enseignant pouvant en outre créer des cours
  • Nouvelle extension pour l'authentification « manuelle », qui empêche les utilisateurs de créer leur accompte eux-mêmes
  • La procédure « mot de passe oublié » utilise une confirmation par courriel avant d'attribuer un nouveau mot de passe
Cours
  • Un bouton placé à côté des activités permet de les cacher ou de les montrer aux étudiants
  • Des semaines ou thèmes entiers peuvent aussi être cachés
  • La liste des praticipants est elle aussi paginée, pour l'utilisation avec de très grandes classes
  • La compression et décompression ZIP est effectuée maintenant par une librairie interne, supprimant la dépendance vis-à-vis de programmes externes. Une conséquence en est que la compression fonctionne maintenant aussi sous Windows
Devoirs
  • Les remises de devoirs peuvent être triées
  • Les courriels de notification de devoir ne sont pas envoyés aux étudiants non inscrits dans le cours
Forums
  • L'affichage des listes des discussions est bien améliorée
  • Les discussions peuvent être déplacées vers un autre forum d'un simple clic
  • La date et l'heure des messages est celle de leur modification, et non plus de leur création
  • Il n'est plus possible d'avoir des messages sans objet ou sans texte
  • Meilleure analyse et affichage du texte des messages
  • Il est maintenant possible de forcer l'envoi des copies des messages du forum « Nouvelles du site » (courriel à tous les étudiants et enseignants actifs)
  • Meilleure gestion des courriels sortant pour éviter l'envoi de doublons, qui pouvait survenir dans de très grandes classes
Module atelier (pas inclus : bientôt disponible en téléchargement séparé)
  • Un nouveau module très intéressant de Ray Kingdon
  • Un outil de grande souplesse offrant un éventail de schémas d'évaluation par les pairs et de fabrication exemples, permettant à un groupe l'examen et l'évaluation de travaux
Tests
  • Nouveau type de question : aléatoire (choisit une question parmi toutes celles d'une catégorie)
  • Nouveau type de question : appariement (les étudiants doivent faire correspondre les réponses avec des questions)
  • Nouveau type de question : appariement aléatoire (crée aléatoirement une question d'appariement avec les questions à réponse courte d'une catégorie)
  • Importation de questions à partir de fichiers texte (concept modulaire permettant l'extension à de nouveaux formats - une module pour le format Blackboard est inclus en version bêta)
  • Un assistant « Créer plusieurs questions » permet de créer rapidement un test aléatoire
  • Les questions d'un test peuvent être mélangées aléatoirement
  • Les réponses d'un test (choix multiples, etc.) peuvent être mélangées aléatoirement
  • Erreur corrigée lorsque le bouton « Enregistrer les notes » n'était pas utilisé
  • La re-notation fonctionne correctement si le test a été modifié
  • La modification d'une question après son utilisation dans un test n'a pas d'influence sur les résultats du test
  • Les réponses des questions à choix multiples sont numérotées par a, b, c et non plus 1, 2, 3
Ressources
  • Meilleure performance lors de la modification d'un long texte ou de ressources HTML
  • Les images sont maintenant affichées centrées dans une page web, et non plus de façon brute
  • Le format genre Wiki a été ajouté comme une nouvelle option pour les pages Ressources
Thèmes personnalisés
  • Un certain nombre de nouveaux styles ont été ajoutés à tous les thèmes fournis en standard. Certains d'entre eux sont ESSENTIELS à l'affichage des nouvelles fonctionnalités. VEUILLEZ LIRE SVP theme/UPGRADE.txt POUR UNE LISTE DES MODIFICATIONS À EFFECTUER SUR LES ANCIENS THÈMES
  • Les thèmes personnalisés peuvent maintenant redéfinir tous les petits boutons graphiques. Il existe pour cela un nouveau paramètre dans theme/x/config.php ($THEME->custompix) et un nouveau dossier facultatif « pix ». Pour un exemple, voyez le thème cordoroyblue.

Les anciennes notes de mise à jour sont disponibles dans le forum des nouvelles de Moodle.org.
